## Key Features

### 1. Type System
- 6 DataObject types with runtime validation (Real, Int, Bool, Categorical, String, Dimension)
- Constraint enforcement (min/max, allowed values)
- JSON serialization/deserialization

### 2. Schema Management
- Deterministic schema hashing (SHA256 of types + constraints)
- Auto-increment schema IDs (schema_001, schema_002, ...)
- Structural compatibility checking with detailed error messages
- Schema registry at .lbp/schema_registry.json

### 3. Dataset Container
- Master storage for experiments, metrics, and arrays
- Comprehensive validation (required params, type checking, constraints)
- Performance metric validation against schema

### 4. LBPAgent Orchestration
- Two-phase initialization resolving circular dependencies
- Schema generation via model introspection
- Hierarchical load/save (Memory → Local → External)
- Full integration with EvaluationSystem and PredictionSystem

### 5. Backward Compatibility
- Legacy decorators remain as aliases (@study_parameter, etc.)
- LBPManager still available and functional
- LocalData supports both study_code and schema_id APIs
- Gradual migration path supported

## Architecture Highlights

### Two-Phase Initialization
Phase 1: Activate models → Initialize features → Generate schema
Phase 2: Create dataset → Attach storage → Initialize systems

### Deterministic Schema IDs
- Same model configuration always generates same schema_id
- Registry ensures consistency across sessions
- Human-readable folder names (schema_001 vs opaque codes)

### Validation Pipeline
1. Static params validated on set_static_values()
2. Experiment params validated on add_experiment()
3. Dimensional params checked for positive integers
4. Performance codes validated against schema
5. Type validation via DataObject.validate()
